## 나의 풀이
### 아이디어
각 변의 길이를 n으로 나눈 몫이 해당 변에 채워질 수 있는 주사위 개수가 되므로,  
가로 몫 * 세로 몫 * 높이 몫이 최대 개수가 된다.  
### 코드
```javascript
function solution(box, n) {
    var answer = 0;
    return box.map(v => Math.trunc(v / n)).reduce((total, v) => total * v, 1);
}
```

## 다른 사람 풀이
```javascript
function solution(box, n) {
    let [width, length, height] = box;

    return Math.floor(width / n) * Math.floor(length / n) * Math.floor(height / n);

}
```
이 풀이를 보니, 굳이 세 개로 정해져있는 배열을 map으로 풀었나 싶어졌다.  
  
```javascript
function solution(box, n) {
    return box.reduce((acc,v) => acc * Math.floor(v / n), 1);
}
```
이 풀이를 보니, 굳이 reduce를 쓸 거면 왜 한번에 계산하지 않고 반복문을 돌렸나 싶어졌다...
